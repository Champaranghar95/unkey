Contributing to Unkey

House Rules
  • Check existing issues and PRs before submitting new ones.
  • Create an issue before beginning development.
  • Pay attention to the needs-approval label on GitHub issues.

For Contributors:

  • Feature Requests / Large-Scale Refactoring: Wait for approval before coding.
  • Bugs, Security, Documentation, etc.: You can start coding immediately.
Developing
  1. Fork and clone the repository.
  2. Create a new branch: git switch -c MY_BRANCH_NAME
  3. Follow the getting started guide.
Installing
  1. Set the correct version of PNPM: corepack enable
  2. Install dependencies: pnpm install
Building and Linting

Build the project: pnpm build

Check code formatting: pnpm fmt

Making a Pull Request
  1. Check "Allow edits from maintainers" when creating your PR.
  2. Link to related issues using refs #XXX or fixes #XXX.
  3. Fill out the PR Template completely.
